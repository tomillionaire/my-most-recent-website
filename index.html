Git and GitHub

What is Git?

Git is a free open source Version Control System that allows developers track versions of their code overtime.

Git tracks our codees by creating snapshots of the code and can be revisited whenever necessary.

Configuring Git

git config --global user.name "Tomillionaire" is used to configure our name on git.

git config --global user.email "aderonmutomilola@gmail.com" is used to configure our email on git.

Basic Commands

k

git init is used to initialize Git in your local repository.

git status is used to check the status of the activities of git in your local repository.

git add "filename" is used to track a single file in git.

git add . is used to track multiple files in git.

git rm --cached "filename" is used to untrack a file in git.

git commit -m "the commit message" is used when we want to save a snapshot of our code. It is like a caption or the description of the changes made to our codes.

git remote add origin "repo url" is used to connect your local repo to github.

git push -u origin master is used to push our code from the local repo to the remote repo.



What is Github?

Github is a web based git repository hosting service.

It is a tool that enables collaboration by hosting shared git repositories that team of developers can all contribute to.

While GitHub uses git, the functionality it provides is entirely different from Git.

Git is a Version Control System while github is an online platform for hosting and sharing code.

Why Use GitHub?

  1. GitHub provides a great way to store your code in a safe and remote location.
  2. Many large open source projects are hosted on Github.
  3. Github encourages collaboration between a team of developers.
